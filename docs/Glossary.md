# Глосарій термінів

## Алгоритми та методи

### Particle Filter (Фільтр частинок)
Метод Монте-Карло для оцінки стану системи. Множина "частинок" представляє ймовірний розподіл стану (позиції БПЛА). Кожна частинка -- це гіпотеза з координатами та вагою. На кожному кроці: пропагація (зсув за рухом), оцінка (порівняння з реальністю), ресемплінг (відкидання поганих гіпотез).

### FAsT-Match (Fast Affine Template Matching)
Алгоритм швидкого пошуку афінного перетворення між шаблоном та зображенням. Замість повного перебору використовує ієрархічний підхід: грубий пошук → звуження простору → точний пошук. Публікація: Korman et al., CVPR 2013.

### KLD-Sampling (Kullback-Leibler Distance Sampling)
Адаптивний метод визначення необхідної кількості частинок. Замість фіксованої кількості, KLD-семплінг збільшує кількість частинок поки розподіл не стане достатньо репрезентативним. Формула: `k = ceil(k / (2*error) * (1 - 2/(9k) + sqrt(2/(9k)) * z)^3)`. Зменшує обчислювальне навантаження при простих розподілах.

### Pearson Correlation (Кореляція Пірсона)
Міра лінійної залежності між двома змінними, від -1 до +1. В контексті проєкту -- нормалізований коефіцієнт кросс-кореляції між зображеннями. Інваріантний до лінійних змін яскравості (фотометрична інваріантність). Реалізація через `cv::matchTemplate(TM_CCOEFF_NORMED)`.

### Photometric Invariance (Фотометрична інваріантність)
Стійкість до змін освітлення між аеро-зображенням та картою. Досягається нормалізацією по середньому та стандартному відхиленню: `score = Σ|xi - (σx/σy)*yi + C|`. Карта може мати інше освітлення, час доби, сезон.

## Параметри алгоритму

### epsilon (ε)
Параметр точності FAsT-Match. Визначає кількість точок семплювання: `n = 10 / ε²`. При ε=0.1 отримуємо 1000 точок. Менше epsilon → більше точок → точніше, але повільніше.

### delta (δ)
Параметр впевненості FAsT-Match. Визначає поріг для відсіювання поганих конфігурацій між рівнями ієрархії. Типове значення: 0.25.

### lowBound (нижня межа кореляції)
Поріг активації для функцій конвертації ймовірності. Кореляції нижче цього значення отримують мінімальну ймовірність. За замовчуванням: 0.2. Задається через CLI `--correlation-bound`.

### quantile (квантиль)
Параметр для KLD-семплінгу. Визначає z-значення з нормального розподілу. При quantile=0.99 система впевнена на 99% що кількість частинок достатня. Значення зчитується з `ztable.data`.

### binSize (розмір біна)
Розмір просторового біна в пікселях для KLD-семплінгу. Частинки в одному біні вважаються еквівалентними. При binSize=5 позиції квантуються до сітки 5x5 пікселів.

### kld_error
Допустима помилка KLD-семплінгу. Менше значення → більше частинок для точнішого представлення розподілу. Типове значення: 0.5.

## Функції конвертації ймовірності

### HPRELU (Half Parametric Rectified Linear Unit)
Кусково-лінійна функція:
- Якщо `lowBound < 0`: `prob = slope * (corr - |lowBound|)` для corr > |lowBound|, інакше 0
- Якщо `lowBound >= 0` і `corr > 0`: `prob = lowBound + corr * (1 - lowBound)`
- Якщо `lowBound >= 0` і `corr <= 0`: `prob = lowBound - |corr| * lowBound`

### GLF (Generalized Logistic Function)
Сигмоїдна функція: `f(x) = A + (K-A) / (C + Q*exp(-B*x))^(1/v)`, нормалізована до `f(1.0)`. Параметр `v` контролюється через `lowBound`. Дає плавний перехід від низьких до високих ймовірностей.

### Softmax
Експоненціальна функція: `prob = exp(correlation)`. Нормалізація відбувається потім у `Particles::normalize()` через ділення на суму всіх ймовірностей.

## Типи співставлення (Match Modes)

### BRISK (Binary Robust Invariant Scalable Keypoints)
Детектор та дескриптор ключових точок. Бінарний дескриптор -- швидкий для порівняння. В проєкті: потребує GPU (`USE_CV_GPU`). Поріг детекції: 20.

### ORB (Oriented FAST and Rotated BRIEF)
Детектор (FAST) та дескриптор (BRIEF) ключових точок з орієнтаційною інваріантністю. Бінарний, швидший за SIFT/SURF. В проєкті: потребує GPU.

## Геопросторові терміни

### GeoTIFF
Формат зображення TIFF з вбудованою геопросторовою інформацією (система координат, проєкція, прив'язка до землі). Використовується для супутникових та аерофотознімків.

### UTM (Universal Transverse Mercator)
Система координат що ділить Землю на 60 зон. Кожна зона має свою проєкцію з координатами Easting (X) та Northing (Y) в метрах. Використовується для конвертації GPS ↔ пікселі через GeographicLib.

### GeoTransform
Масив з 6 коефіцієнтів GDAL що описує прив'язку пікселів до координат:
```
X_geo = GT[0] + pixel*GT[1] + line*GT[2]
Y_geo = GT[3] + pixel*GT[4] + line*GT[5]
```

### SVO (Semi-direct Visual Odometry)
Метод візуальної одометрії -- оцінка руху камери з послідовності зображень. В наборі даних SVO-позиція записана в `SvoX/Y/Z`. Використовується для обчислення вектора руху між кадрами.

## Математичні терміни

### Affine Transformation (Афінне перетворення)
Геометричне перетворення що зберігає паралельність ліній: комбінація трансляції, обертання, масштабування та зсуву. В 2D описується матрицею 2x3. В проєкті параметризується через `MatchConfig(tx, ty, r1, r2, sx, sy)`.

### Configuration Space (Простір конфігурацій)
6-вимірний простір можливих афінних перетворень: 2 трансляції + 2 обертання + 2 масштаби. FAsT-Match ієрархічно звужує цей простір. Для частинок: кожна частинка генерує `scale_steps² * rotation_steps * r2_steps` конфігурацій.

### Quaternion (Кватерніон)
Спосіб представлення обертання в 3D через 4 числа (x, y, z, w). Уникає проблеми gimbal lock. В проєкті конвертується в кути Ейлера (Roll/Pitch/Yaw) для отримання напрямку БПЛА.

### Z-table (Z-таблиця)
Таблиця значень кумулятивної функції стандартного нормального розподілу. Зберігається в `ztable.data`. Використовується KLD-семплінгом для визначення порогу кількості частинок при заданому рівні впевненості.
